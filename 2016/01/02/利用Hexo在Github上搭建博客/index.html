利用Hexo在Github上搭建博客

发表于 2016-01-02 | 分类于 技术
元旦假期终于可以静下来休息一下了，回忆过去的一年里，除了年龄的增长我还有什么收获？一时间竟然毫无头绪。

元旦假期终于可以静下来休息一下了，回忆过去的一年里，除了年龄的增长我还有什么收获？一时间竟然毫无头绪。人生阅历的积累并不是简单的叠加，在记忆的程序里外界的事务源源不断的进入，其中一部分则随着时间的更迭，慢慢被遗忘。于是乎一些事情当我回想起来时，早就已经被抛置于九霄云外，再也找不回来。唯一能留住一点这些被遗忘的记忆最好的方式就是在被遗忘前把它们记录下来。登录微博、QQ、微信才突然发现这些应用所能记录的都只是些只言片语，要是稍微大一点就要在一片花花绿绿的背景里使用反人类的工具“生成”一篇文章，所以才有了搭建一个自己博客的想法，作为对过去的记录，对现在的礼物，对未来的帮助。

平台选择

写博客的人，会经历三个阶段

  • 第一阶段，刚接触Blog，觉得很新鲜，试着选择一个免费空间来写。
  • 第二阶段，发现免费空间限制太多，就自己购买域名和空间，搭建独立博客。
  • 第三阶段，觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。

本着不折腾不舒服的原则，果断选择了Github+Hexo的方案。Github号称编程界的Facebook。作为世界上最大的同性交友网站，简单说，它是一个具有版本管理功能的代码仓库。Github设计了Pages功能，允许用户自定义项目首页，用来替代默认的源码列表。Hexo是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github上。作者是来自台湾的tommy351。引用tommy351的话，Hexo 是快速、简单且功能强大的 Node.js 博客框架(A fast, simple & powerful blog framework, powered by Node.js.)。

Github+Hexo使用

网上关于GitHub的教程多如牛毛，在Windows上命令行的客户端还是很好用的。Hexo的详细介绍也是一大堆一看就明白是怎么回事。贴上几个备用。

  • 如何在 GitHub 上写博客
  • 通过Hexo在Github上搭建博客教程
  • hexo系列教程

Hexo主题选择

作为一个个人博客来说，除了内容以外，我觉得最为重要的是长相（毕竟这是一个看脸的时代）。所以在写完文章后，还要给它来个漂亮的衣服，秀一下气质。
网上搜一下有那些好看的 Hexo 主题？，要求不多，只要显示文章内容就好了啦（最好有社交啦，字体好看点，排版优美点，颜色搭配舒服点，整体大气点，访问速度快一点，动画特效多一点）。于是发现NexT.Mist颜值很高，还有详细的使用说明。而且拥有响应式布局设计。
选好之后生成博客，接下来就是见证奇迹改造的时刻，“颜值再高，也要动刀”，来个微整形，改造下配色，修改下字体大小，换个头像。
防止遗忘，贴一下改造文件记录

  • 修改文件hexo\themes\next\source\css\_schemes\Mist\_header.styl
  • 修改文件hexo\themes\next\source\css\_schemes\Mist\_posts-expanded.styl
  • 修改文件hexo\themes\next\source\css\_schemes\Mist\_menu.styl
  • 修改文件hexo\themes\next\source\css\_common\_section\sidebar.styl
  • 修改文件hexo\themes\next\source\css\_common\_component\back-to-top.styl

大功告成！
